## Прогресс относительно README.md и backend/PLAN.md
- WebSocket API теперь обслуживает полный набор фаз: подключение проходит через сообщение `join`, далее клиент получает тики таймера, отчеты по завершению фаз и может асинхронно отправлять действия (ставки, производство, займы, стройки или `skip`) без блокировки таймера.
- Реализована механика старшинства из README: при создании `GameSession` бросается `1d6` с повторными бросками при ничьей, фиксируется стартовый порядок и ведется журнал бросков; в `end_month` публикуется история ротаций на следующий месяц.
- Каждая фаза теперь ограничена таймером на 60 секунд, реализованным в `game_logic` (`PhaseTimer`). После завершения фазы публикуется структурированный отчет с журналом действий и аналитикой по игрокам, что покрывает требования backend/PLAN.md.
